我的世界服务器本地搭建指南第一期

Ebores · 2025-3-31 · 次阅读

服务器搭建指南第一期（简单粗暴版）

本地部署和云端部署区别

本地部署

  1. 通过局域网连接
  2. 稳定性差 容易宕机
  3. 一次性投入 费用低
    本地部署只需要一次性投入设备成本，就可以持续使用，但稳定性受限于家用网络

云端部署

  1. 互联网广播连接
  2. 稳定性高 维护方便
  3. 费用较高
    云端部署是一种相对简单快速的解决方案，但每月的租赁费用并不便宜
    各位小伙伴可以依照自己的情况选择部署方案，我们暂时以本地部署为例

开始本地部署

首先要在服务器上安装Java，对于1.17以上的游戏版本，需要Java SE16及更高的Java支持

而1.17及以下可以考虑更低版本的Java

所有的安装包都可以在下方的官网上找到

Java官网: (Oracle | Cloud Applications and Cloud Platform)

下载演示

1 .访问官网



2 .点击product



3 .点击Java



4 .下拉网页



5 .点击Download Java now



6 .下拉页面下载Java



现在配置好了Java环境，接下来我们就开始部署服务端

部署服务端

下面是几类主流的我的世界服务端核心





  1. Spigot服务端 （适配要求适中 较稳定 支持安装插件 不支持安装MOD）适用对外开放的服务器
  2. Paper服务端 （适配要求低 稳定性差 支持安装插件 不支持安装MOD）适用低配置的开放服务器

  3. 官方服务端 （稳定性最高 漏洞少 不支持安装插件 不支持安装MOD）适用朋友间的原版联机

    下载链接：

    1 .SpigotMC - High Performance Minecraft Community

    2 .Home | PaperMC

    3 .https://www.minecraft.net/

    本次教程以spigot服务端为例

下载spigot服务端

  • 注意版本相应信息


  • 点击Continue


  • 下拉点击此处下载


服务器核心安装

在安装服务器核心前，我们需要用到一个工具，它叫 Git，可以直接点下面链接去下载

Git官网链接: Git - Downloads

  • 进到官网以后，点击Download


  • 选择64-bit Git for Windows Setup


  • Git安装完后，回到刚刚下载spigot服务端的文件夹
  • 右键文件夹，就会出现一个Git Bash Here，然后我们单机它，就出现了一个窗口


  • 在这个窗口中输入以下命令，然后回车，它就会开始自动构建（大概十几分钟）
  • java -jar BuildTools.jar


如果中途报错或者没有成功地执行下去，你可以重新键入这个命令

服务器核心的启动

  • 这里已经构建好了，这边出现了一个spigot-1.17.jar文件，这个就是我们的服务器核心，


  • 我们创建一个新文件夹（我用server做例子）然后把核心拖进去


  • 在这个文件夹中新建一个文本文档，将后缀名改为bat


  • 右键编辑这个文件或者以文本文档的方式打开输入命令，然后保存关闭，运行此文件
    java -jar spigot-1.17.jar （按需修改）


  • 在运行中会突然跳掉，因为需要同意一个协议，在eula.txt文件里
    将false修改为true，记得保存


  • 保存后，再重新启动start.bat，以后可以通过它来启动我们的服务器
    看到这个弹窗出现就说明你已经成功地将服务器启动了
